# Portfolio Reflection

## What was the problem you were solving in the projects for this course?

The problem I was solving was how to organize and retrieve course information efficiently for academic advisors. The projects required designing a system that could store course data, display a sorted list of courses, and allow users to search for a specific course and view its prerequisites.

## How did you approach the problem? Consider why data structures are important to understand.

I approached the problem by evaluating different data structures and selecting a binary search tree because it provides efficient searching and naturally maintains data in sorted order. Understanding data structures is important because they affect the speed, memory usage, and overall efficiency of software applications.

## How did you overcome any roadblocks you encountered while going through the activities or project?

One challenge I encountered was debugging issues related to file access, function definitions, and compilation errors. I overcame these problems by carefully reviewing my code, testing each feature individually, and using the Eclipse debugger and build tools to identify errors.

## How has your work on this project expanded your approach to designing software and developing programs?

This project helped me understand the importance of planning before coding. I learned how selecting the proper data structure can simplify algorithms and improve performance. I also gained more confidence in breaking larger problems into smaller functions.

## How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

This project taught me to use meaningful variable names, comments, and modular functions to improve readability. I now place greater emphasis on writing code that can be easily updated and maintained in the future.
